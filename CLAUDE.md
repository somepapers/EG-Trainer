# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## How to run

Open `index.html` directly in a browser. No build step, no dependencies, no server needed.

## Architecture

Single-file static HTML application — all CSS and JS are embedded in [index.html](index.html).

### Domain: Rubik's Cube EG-1 trainer

EG-1 is a 2x2 Rubik's Cube method that solves the first face + permutation in one algorithm. This tool generates scrambles (inverted algorithms), tracks correctness stats, and shows the correct formula when the user gets one wrong.

### Data model

Two parallel data structures per case type (H, Pi, T, L, U, S, AS):

- **`<Case>_cases`** — arrays of sub-case objects, each with `name` (e.g. "H1") and `algs` (array of algorithm strings in standard Rubik's notation: R, U, F, L with ', 2 modifiers). Used to generate scrambles.
- **`<Case>_show_correct`** — arrays of strings, indexed by sub-case number (0-based), holding the "display" formula shown on wrong answers. These are a curated subset of the algorithms above presented as the "correct" answer.

The scramble is generated by `invertAlg()` — reversing the alg string and flipping direction (e.g. `R` → `R'`, `R'` → `R`, `R2` stays `R2`).

### Key functions

| Function | Purpose |
|---|---|
| `nextScramble()` | Picks a random checked case type, then a random sub-case and algorithm. Avoids repeating the same sub-case within 20 tries. Displays the inverted alg as the scramble. |
| `correct()` | Increments correct count, saves to localStorage, shows "蒸蚌", auto-advances after 2.5s. |
| `wrong()` | Increments wrong count, saves, and shows the correct display formula from `<Case>_show_correct` based on the last sub-case name. |
| `invertAlg(alg)` | Reverses and inverts a Rubik's cube algorithm to produce the scramble. |
| `updateStats()` | Renders the stats table from `statsCase` object. |
| `showPage(n)` | Toggles between page 1 (training) and page 2 (statistics). |

### State

- Stats persisted in `localStorage` under key `"eg1_case"` as JSON `{H: {correct, wrong}, Pi: {...}, ...}`.
- `last<Case>Case` variables track the most recent sub-case name per case type (used to look up the correct display formula).
- `currentAlg`, `currentCase`, `currentScramble` hold the active training round state.
